# 爱宝贝儿后端API对接文档

## 1. 文档概述

本文档详细描述了爱宝贝儿商城后端API接口，分为两个前端对接：

| 前端项目 | API前缀 | 说明 |
|----------|---------|------|
| **小程序端** | `/api/` | 面向C端用户，功能包括登录、购物、订单等 |
| **后端管理系统** | `/api/admin/` | 面向管理员，功能包括用户管理、订单统计、仪表盘等 |

### 技术栈
- **框架**: Spring Boot 2.7.x
- **数据库**: MySQL 8.0+
- **ORM**: MyBatis Plus 3.5.x
- **认证**: JWT (JSON Web Token)
- **构建工具**: Maven

### 服务地址
```
http://localhost:8080
```

---

## 2. 通用说明

### 2.1 请求头格式

**所有接口**（除登录/注册外）需要在请求头中携带Token：

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 2.2 统一响应格式

**成功响应**:
```json
{
    "code": 200,
    "message": "success",
    "data": {...}
}
```

**失败响应**:
```json
{
    "code": 400,
    "message": "错误信息描述"
}
```

### 2.3 状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权（Token无效或过期） |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 2.4 时间范围参数

部分统计接口支持时间范围筛选：

| 参数值 | 说明 |
|--------|------|
| `7d` | 近7天 |
| `30d` | 近30天 |
| `90d` | 近90天 |

## 3. 小程序端 API

### 模块 1：用户认证

#### 1.1 用户登录
```
POST /api/auth/login
Content-Type: application/json
```

**请求参数**:
| 参数名 | 类型 | 必须 | 说明 |
|--------|------|------|------|
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |

**请求示例**:
```json
{
    "username": "test001",
    "password": "password123"
}
```

**响应示例**:
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "user": {
            "id": 1,
            "username": "test001",
            "nickname": "宝宝妈妈",
            "avatar": "https://example.com/avatar.jpg",
            "phone": "13800138001",
```

## 5. 错误码详解

### 5.1 通用错误码

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 检查请求参数是否正确 |
| 401 | 未授权 | Token无效或过期，请重新登录 |
| 403 | 禁止访问 | 无权限访问该资源 |
| 404 | 资源不存在 | 检查请求的资源ID是否正确 |
| 500 | 服务器内部错误 | 联系后端开发人员 |

### 5.2 业务错误码

| 错误信息 | 说明 |
|----------|------|
| 请先登录 | 用户未登录或Token已过期 |
| 用户不存在 | 用户ID错误或用户已删除 |
| 商品不存在 | 商品ID错误或商品已下架 |
| 库存不足 | 商品库存不够当前购买数量 |
| 订单不存在 | 订单ID错误或订单已删除 |
| 订单状态不允许此操作 | 当前订单状态不支持该操作 |
| 已收藏过该商品 | 该商品已在收藏夹中 |
| 收货地址不存在 | 地址ID错误或地址已删除 |
| 余额不足 | 用户余额不够支付订单 |
| 退款申请已处理 | 退款申请不能重复处理 |

## 7. 注意事项

### 7.1 前端对接注意事项

1. **Token存储**
   - 建议使用 `localStorage` 或 `uni.setStorageSync()` 存储Token
   - 退出登录时务必清除Token

2. **请求拦截**
   - 建议封装统一的请求方法，自动携带Token
   - 遇到401响应时，自动跳转登录页

3. **参数校验**
   - 前端需对必填参数进行校验
   - 日期格式：`yyyy-MM-dd`
   - 时间格式：`yyyy-MM-dd HH:mm:ss`

4. **分页处理**
   - 默认页码：1
   - 默认每页数量：10或20
   - 分页响应包含 `total`（总记录数）、`page`（当前页）、`pageSize`（每页数量）

### 7.2 小程序端注意事项

1. **登录流程**
   - 首次打开小程序尝试静默登录
   - 失败则跳转登录页面引导用户登录

2. **购物车同步**
   - 添加购物车后及时刷新购物车数据
   - 下单后清空已购买的购物车商品

3. **订单状态流转**
   - 待付款 → 已付款 → 已发货 → 已完成
   - 可中途取消（待付款/已付款状态）
   - 可申请退款（已付款/已发货状态）

### 7.3 后端管理系统注意事项

1. **权限控制**
   - 只有管理员才能访问 `/api/admin/` 开头的接口
   - 建议后端对管理员账号进行单独标识

2. **数据安全**
   - 敏感操作建议记录操作日志
   - 批量删除操作建议进行二次确认

3. **统计报表**
   - 时间范围筛选采用左闭右闭区间
   - 统计数据建议添加缓存机制
