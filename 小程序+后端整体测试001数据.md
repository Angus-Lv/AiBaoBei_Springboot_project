
## 7. 数据验证规则

### 7.1 预约数据验证

| 字段 | 类型 |  必填 | 验证规则 |
|------|------|------|----------|
| serviceType | string | 是 | 必须为 bath、playground、photo 或 hair 之一 |
| date | string | 是 | 格式为 YYYY-MM-DD，且必须是未来日期 |
| time | string | 是 | 格式为 HH:00-HH:00，且必须在服务开放时间内 |
| babyName | string | 是 | 非空字符串 |
| babyAge | string | 是 | 非空字符串 |
| contactName | string | 是 | 非空字符串 |
| contactPhone | string | 是 | 11位数字 |

### 7.2 商品数据验证

| 字段 | 类型 |  必填 | 验证规则 |
|------|------|------|----------|
| id | number | 是 | 商品ID |
| name | string | 是 | 商品名称 |
| price | string | 是 | 价格 |
| image | string | 是 | 商品图片 |
| stock | number | 是 | 库存数量 |

## 8. 注意事项

1. 所有API请求均使用HTTPS协议
2. 响应数据的code字段为200表示成功，其他值表示失败
3. 图片资源使用相对路径或完整URL
4. 服务类型字段必须严格按照规定值传递
5. 日期格式必须为YYYY-MM-DD
6. 时段格式必须为HH:00-HH:00
7. 手机号码必须为11位数字

## 9. 测试环境

- 前端地址: http://localhost:8080
- 后端地址: http://localhost:3000/api
- 数据库: MySQL
- 缓存: Redis
