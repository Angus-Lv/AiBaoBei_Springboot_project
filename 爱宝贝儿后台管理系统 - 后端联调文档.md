# 爱宝贝儿后台管理系统 - 后端联调文档

## 1. 项目概述

本文档旨在为前端与后端的联调提供详细的指导，包括前端页面的修改内容、API调用路径、数据结构以及联调注意事项。

## 2. 技术栈

- **前端**：Vue 3 + Composition API + Element Plus + Axios
- **后端**：基于Spring Boot的RESTful API
- **API前缀**：`/api/admin`
- **基础URL**：`http://localhost:8080`

## 5. 联调注意事项

### 5.1 API调用格式

- **请求方法**：遵循RESTful规范，GET/POST/PUT/DELETE
- **请求参数**：查询参数使用URL查询字符串，请求体使用JSON格式
- **响应格式**：统一为JSON格式，包含code、message、data字段
  ```json
  {
    "code": 200,
    "message": "操作成功",
    "data": {
      "list": [],
      "total": 100,
      "page": 1,
      "pageSize": 20
    }
  }
  ```

### 5.2 错误处理

- **前端**：统一处理API错误，显示友好的错误提示
- **后端**：返回标准的错误码和错误信息
  | 错误码 | 描述 |
  | --- | --- |
  | 200 | 成功 |
  | 400 | 请求参数错误 |
  | 401 | 未授权 |
  | 403 | 禁止访问 |
  | 404 | 资源不存在 |
  | 500 | 服务器内部错误 |

### 5.3 认证信息

- **登录账号**：admin
- **登录密码**：admin123
- **认证方式**：预留JWT但使用模拟token
- **请求头**：不需要Authorization头（模拟token，方便调试）

### 5.4 测试数据

- **分类数据**：9个分类（全部、奶粉、尿裤、服装、玩具、喂养、护理、营养、出行）
- **商品数据**：16个商品，覆盖所有分类

## 6. 调试建议

1. **使用Postman**：先通过Postman测试API接口是否正常
2. **开启前端调试**：使用浏览器开发者工具查看网络请求
3. **查看后端日志**：了解API调用情况和错误信息
4. **数据一致性**：确保前端发送的数据格式与后端要求一致
5. **错误处理**：测试各种错误场景，确保前端能正确处理
6. **分页处理**：注意分页参数的传递和处理
7. **时间格式**：统一使用`YYYY-MM-DD HH:mm:ss`格式

## 7. 通用接口

### 7.1 上传文件
- **接口路径**: `/api/admin/upload`
- **请求方法**: POST
- **请求参数**:
  | 参数名 | 类型 | 必填 | 描述 |
  | --- | --- | --- | --- |
  | file | file | 是 | 文件 |
  | type | string | 否 | 文件类型 (image/file) |

- **响应数据**:
  ```json
  {
    "code": 200,
    "message": "上传成功",
    "data": {
      "url": "https://example.com/uploads/file.jpg"
    }
  }
  ```

### 7.2 获取统计数据
- **接口路径**: `/api/admin/statistics`
- **请求方法**: GET
- **请求参数**:
  | 参数名 | 类型 | 必填 | 描述 |
  | --- | --- | --- | --- |
  | type | string | 是 | 统计类型 (sales/users/orders) |
  | startDate | string | 否 | 开始日期 |
  | endDate | string | 否 | 结束日期 |

- **响应数据**:
  ```json
  {
    "code": 200,
    "message": "获取成功",
    "data": {
      "total": 10000,
      "data": [
        { "date": "2024-01-01", "value": 1000 },
        { "date": "2024-01-02", "value": 1200 }
      ]
    }
  }
  ```
